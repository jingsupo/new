数据库表分析

根据项目需求，可以分析项目中有以下几张表：

  • User表：保存用户信息
  • House表：保存房屋信息
  • Order表：保存用户订单信息
  • Area表：保存区域信息
  • Facility表：保存房屋设施信息
  • HouseFacility表：房屋与设备多对多外键关联
  • HouseImage表：保存房屋信息中的图片数据

User表

字段名 说明 备注
id 用户编号 主键
name 用户昵称
password_hash 加密的密码
mobile 手机号
real_name 真实姓名
id_card 身份证号
avatar_url 用户头像路径
  • 其对应模型中定义以下属性：
    • houses: 标识该用户的所有房屋，与 House 表产生关联
    • orders: 标识该用户订单，与 Order 表产生关联

Area表

字段名 说明 备注
id 区域编号 主键
name 区域名字
  • 其对应模型中定义以下属性：
    • houses: 标识该区域对应的所有房屋，与 House 表产生关联

House表

字段名 说明 备注
id 房屋编号 主键
avatar_url 用户头像路径
user_id 房屋主人的用户编号 外键，关联 user 表中的 id
area_id 归属地的区域编号 外键，关联 Area 表中的id
title 标题
price 单价 单位：分
address 地址
room_count 房间数目
acreage 房屋面积
unit 房屋单元， 如几室几厅
capacity 房屋容纳的人数
beds 房屋床铺配置
deposit 房屋押金
min_days 最少入住天数
max_days j最多入住天数 0 表示不限制
order_count 预订完成的该房屋的订单数
index_image_url 房屋主图片的路径
  • 其对应模型中定义以下属性：
    • facilities：标识该房屋所有的这设施，与 HouseFacility 表产生关联
    • images：标识该房屋所有的图片，与 HouseImage 表产生关联
    • orders：标识该房屋所有的订单，与 Order 表产生关联

Facility 表

字段名 说明 备注
id 设施编号 主键
name 设施名字

HouseFacility 表

字段名 说明 备注
house_id 房屋编号 主键，外键关联于 House 表中的 id
facility_id 房屋编号 主键，外键关联于 Facility 表中的 id

Order 表

字段名 说明 备注
id 订单编号 主键
user_id 下单用户的id 外键，关联 User 表中的 id
house_id 预订房屋的id 外键，关联 House 表中的 id
begin_date 预订的起始时间
end_date 预订的结束时间
days 预订总天数
house_price 房屋的单价
amount 订单总金额
status 订单的状态
comment 订单的评论/拒单原因

HouseImage 表

字段名 说明 备注
id 图片编号 主键
house_id 房屋编号 外键，关联于 House 表中的 id
url 图片路径
